# NO-GO Critic Blockers Loop v1

| risk | why | required fix | required test | blocker |
|---|---|---|---|---|
| Parallel implementation false confidence | Replay/state-processor can pass without exercising production reducers, pollers, recovery logic, or render boundaries. | Make `--mode sut` execute through `PiHarnessSutAdapter`; prevent SUT semantic frames from being derived only from replay model state. | Running-agent fixture in SUT mode executes real extension code, records production UI/poller calls, and fails if the extension module is replaced with a no-op. | Confirm `@gaodes/pi-test-harness` API/lifecycle compatibility and valid SUT extension/module paths. |
| Incomplete same-timestamp ordering | Ordering is underspecified for fixture/import/fs/artifact/timer/poll/notification/render events. | Define canonical ordering; add stable sequence IDs; record ordered processed event IDs on every frame. | Mixed same-timestamp fixture produces the documented byte-identical frame trace across 100 repeated runs. | Plan must choose one authoritative ordering contract before scheduler changes. |
| Wall-clock and async nondeterminism | `Date.now`, native timers, async pollers, globals, and leftover in-flight work can leak nondeterminism. | Inject fake clock into production poller; add native timer leakage detection; add poller/global cleanup; enforce max-step, max-duration, and zero-delay loop guards. | 100 in-process and 100 fresh-process runs produce identical JSON/text; intentional zero-delay timer loop fails with documented guard error. | Production polling code must accept injected clock or expose a safe test boundary. |
| Fixture/artifact safety gap | Imports may mutate sources, escape fixture roots, leak secrets, or preserve machine-specific paths. | Sandbox-copy imports; reject traversal/absolute paths by default; enforce source immutability; redact secrets/home paths; emit manifest with hashes and redaction status. | Traversal fixture is rejected, secret fixture is redacted, manifest records redaction, and pre/post source hashes are unchanged. | Import format and allowed path policy must be finalized. |
| PTY/PNG/process coverage overclaim | Optional or unavailable dependencies can be mistaken for terminal conformance or screenshot support. | Mark capabilities as implemented/stubbed/optional/unavailable; make `doctor` run operational PTY/PNG probes; split semantic CI from optional conformance CI. | Without deps, `doctor` reports PTY/PNG unavailable and only conformance suite is skipped/failed; with deps, real PTY spawn and PNG render probes pass before marking operational. | Decide whether PTY conformance and PNG generation are first-delivery scope; ensure CI deps if kept. |
